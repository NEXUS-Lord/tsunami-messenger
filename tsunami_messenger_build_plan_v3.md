## 🧠 Game Concept

You are a motorcycle courier in a low-poly Japanese coastal city. A tsunami warning has just triggered. You have **3 minutes** to deliver evacuation notices to marked locations before the wave hits. The city slowly floods as you ride. Other players appear as ghost couriers in the same world.

**One-line pitch:** *"Motorcycle courier racing a tsunami through low-poly Japan."*

**Win:** Deliver all notices before the wave engulfs the city.
**Lose:** Time runs out or water level reaches your bike.

## ⚠️ KNOWN BUGS FROM V1 — PRE-FIXED IN EVERY PROMPT BELOW

These bugs were discovered and fixed in the first build. Every prompt below already has the fix baked in:

1. **Inverted controls** — W was moving backward. Fix: movement Z uses `-= Math.cos(rotation.y) * speed` NOT `+=`
2. **Camera looking straight down** — looked like a tank. Fix: camera offset is BEHIND the bike using negated sin/cos, low angle
3. **Water instant game-over** — water started at y=-0.5, too close to ground. Fix: always start at `CONFIG.WATER_START_Y` (-5)
4. **No building collision** — bike drove through walls. Fix: `window.CITY_BUILDINGS` array in city.js, AABB check in player.js
5. **Turn speed too fast** — bike spinning 360s. Fix: `PLAYER_TURN_SPEED: 0.015` in config

```
MOTORCYCLE MESH (make it look good with primitives):
Build a THREE.Group containing:

- Seat/fairing: BoxGeometry(1.0, 0.35, 0.85) slightly darker orange, positioned at top-rear of body
- Engine block: BoxGeometry(0.7, 0.4, 0.7), color 0x555555, center of body
- Exhaust pipe: CylinderGeometry(0.06, 0.08, 1.2, 6), color 0x888888, along right side
- Front fork: BoxGeometry(0.12, 1.0, 0.12), color 0x666666, angled forward
- Front wheel: CylinderGeometry(0.42, 0.42, 0.22, 12) rotated Z 90deg, color CONFIG.COLORS.playerDetail
- Back wheel: same, rear position
- Handlebar: BoxGeometry(1.3, 0.08, 0.08), color 0x999999, at top-front
- Headlight: BoxGeometry(0.25, 0.2, 0.1), MeshLambertMaterial emissive 0xFFFFAA, emissiveIntensity 2.0 — front of bike
- Tail light: BoxGeometry(0.3, 0.15, 0.08), MeshLambertMaterial emissive 0xFF0000, emissiveIntensity 2.0 — rear
- Small PointLight(0xFFFFAA, 1.5, 18) attached to front — headlight cone effect
Group initial position: y = 0.52
Add group to scene.

PHYSICS:
- Internal: speed = 0
- W/ArrowUp: speed += 0.018, capped at CONFIG.PLAYER_SPEED
- S/ArrowDown: speed -= 0.012, capped at -CONFIG.PLAYER_SPEED * 0.4
- No input: speed *= 0.88 (friction), snap to 0 if abs < 0.001
- A/ArrowLeft: if abs(speed) > 0.01, mesh.rotation.y += CONFIG.PLAYER_TURN_SPEED * (speed / CONFIG.PLAYER_SPEED)
- D/ArrowRight: if abs(speed) > 0.01, mesh.rotation.y -= CONFIG.PLAYER_TURN_SPEED * (speed / CONFIG.PLAYER_SPEED)

MOVEMENT (CRITICAL — v1 had this inverted, do it correctly):
- sinY = Math.sin(mesh.rotation.y)
- cosY = Math.cos(mesh.rotation.y)
- newX = mesh.position.x + sinY * speed
- newZ = mesh.position.z - cosY * speed   ← NEGATIVE cosY. This is correct. Do not change it.
- Clamp newX and newZ within ±CONFIG.CITY_SIZE/2

BUILDING COLLISION (CRITICAL — do not skip):
Before applying newX/newZ, check window.CITY_BUILDINGS array:
function collidesWithBuilding(nx, nz) {
  var buildings = window.CITY_BUILDINGS;
  if (!buildings || !buildings.length) return false;
  for (var i = 0; i < buildings.length; i++) {
    var b = buildings[i];
    if (nx > b.minX - 0.8 && nx < b.maxX + 0.8 && nz > b.minZ - 0.8 && nz < b.maxZ + 0.8) return true;
  }
  return false;
}

Apply movement with wall-sliding:
- If !collidesWithBuilding(newX, newZ): apply both
- Else try X only: if !collidesWithBuilding(newX, mesh.position.z): apply X only
- Else try Z only: if !collidesWithBuilding(mesh.position.x, newZ): apply Z only
- Else: neither (blocked)

VISUAL DETAILS:
- Wheel rotation: frontWheel.rotation.y += speed * 2.5 each frame
- Bike lean: mesh.rotation.z = -speed * 0.08 when turning left/right (subtle lean)
- Headlight PointLight position follows front of bike each frame

update(delta, keys, waterLevel):
- Run all physics above
- if mesh.position.y <= waterLevel + 0.3: this.drowned = true

checkDelivery(deliveryPoints):
- Return index i if distance3D(mesh.position, deliveryPoints[i].mesh.position) < 3.5
- Else return -1

Output ONLY the JavaScript code for player.js. Stop after the last closing brace.
Do NOT generate any other file. Stop and wait.
```

```
function updateCamera() {

  var behindX = player.mesh.position.x - Math.sin(angle) * 9
```
